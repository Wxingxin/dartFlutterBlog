# 一、布局组件（Layout Widgets）

👉 **核心职责：排列 / 分配空间 / 定位**

## ① 线性布局（主轴 / 副轴）

| 组件           | 作用               |
| ------------ | ---------------- |
| **Row**      | 横向排列             |
| **Column**   | 纵向排列             |
| **Flex**     | Row / Column 的底层 |
| **Expanded** | 占满剩余空间           |
| **Flexible** | 可伸缩占比            |
| **Spacer**   | 空白占位（Flex 专用）    |

📌 场景：表单、工具栏、列表项

---

## ② 叠放 & 定位布局（Z 轴）

| 组件                  | 作用            |
| ------------------- | ------------- |
| **Stack**           | 子组件叠放         |
| **Positioned**      | 绝对定位（Stack 内） |
| **Positioned.fill** | 填满 Stack      |

📌 场景：角标、浮层、背景图

---

## ③ 对齐类布局（位置对齐）

| 组件         | 作用           |
| ---------- | ------------ |
| **Align**  | 精确对齐         |
| **Center** | 居中（Align 特化） |

📌 场景：居中 loading / 空状态

---

## ④ 流式 / 自适应布局

| 组件                | 作用   |
| ----------------- | ---- |
| **Wrap**          | 自动换行 |
| **LayoutBuilder** | 约束感知 |
| **MediaQuery**    | 屏幕信息 |

📌 场景：响应式、标签流

---

## ⑤ 表格 / 网格（非滚动）

| 组件           | 作用   |
| ------------ | ---- |
| **Table**    | 表格布局 |
| **TableRow** | 表格行  |

📌 场景：数据对齐展示

---

## ⑥ Sliver 布局（滚动体系）

| 组件                      | 作用                 |
| ----------------------- | ------------------ |
| **SliverFillRemaining** | 填满剩余滚动区域           |
| **SliverPadding**       | Sliver padding     |
| **SliverToBoxAdapter**  | 普通 Widget 转 Sliver |

📌 场景：复杂滚动页面

---

# 二、容器组件（Container / Box Widgets）

👉 **核心职责：尺寸 + 装饰 + 约束**

## ① 通用容器

| 组件                       | 作用    |
| ------------------------ | ----- |
| **Container**            | 万能盒子  |
| **SizedBox**             | 固定尺寸  |
| **ConstrainedBox**       | 约束尺寸  |
| **UnconstrainedBox**     | 解除约束  |
| **FractionallySizedBox** | 百分比尺寸 |
| **AspectRatio**          | 固定比例  |

---

## ② 间距 / 边距容器

| 组件           | 作用                 |
| ------------ | ------------------ |
| **Padding**  | 内边距                |
| **SizedBox** | 间距（width / height） |

---

## ③ 装饰 & 外观容器

| 组件               | 作用   |
| ---------------- | ---- |
| **DecoratedBox** | 装饰   |
| **ClipRRect**    | 圆角裁剪 |
| **ClipOval**     | 圆形裁剪 |
| **Opacity**      | 透明度  |
| **Transform**    | 变换   |

---

## ④ 约束 & 尺寸感知

| 组件                  | 作用     |
| ------------------- | ------ |
| **IntrinsicWidth**  | 子组件决定宽 |
| **IntrinsicHeight** | 子组件决定高 |

⚠️ 性能开销大，只在必要时用

---

# 三、一个「一句话区分」的方法（非常好用）

> **问自己一句话：**

* “我要**怎么排子组件**？” → **布局组件**
* “我要这个盒子**长什么样**？” → **容器组件**

---

# 四、常见混淆点（直接帮你澄清）

### ❓ Container 是布局组件吗？

❌ 不是
✅ 它是 **装饰 + 尺寸容器**

---

### ❓ Padding 是布局还是容器？

👉 **容器组件**
它不负责排列兄弟，只影响自己 child

---

### ❓ Align / Center 呢？

👉 **布局组件**
它们决定 child 的位置

---

## 五、学习顺序建议（给你实话）

### 必须吃透（第一优先级）

* Row / Column
* Expanded / Flexible
* Stack / Positioned
* SizedBox / Padding
* Align / Center

### 第二阶段

* Wrap
* LayoutBuilder
* AspectRatio / FractionallySizedBox
